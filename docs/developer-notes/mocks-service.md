Alasan kita menggunakan kombinasi `async` dan `Promise` pada service untuk mengambil data dari source of truth adalah karena pada aplikasi nyata, data biasanya tidak berasal dari file lokal melainkan dari sumber eksternal seperti API, server, atau database yang membutuhkan waktu untuk merespons. Operasi tersebut bersifat asynchronous sehingga JavaScript tidak perlu menghentikan seluruh eksekusi program sambil menunggu data diterima. `Promise` digunakan untuk merepresentasikan pekerjaan yang belum selesai dan akan menghasilkan data di masa depan, sedangkan `async/await` memberikan cara penulisan yang lebih mudah dibaca karena alurnya terlihat seperti kode synchronous atau berurutan. Pada tahap development, meskipun data masih berasal dari file mock JSON lokal, service tetap dibuat menggunakan `async` dan `Promise` untuk mensimulasikan perilaku API sungguhan, termasuk proses menunggu respons, menampilkan loading state, dan menjaga agar arsitektur aplikasi tetap konsisten ketika nantinya source of truth dipindahkan dari mock data ke backend atau layanan API yang sebenarnya.
